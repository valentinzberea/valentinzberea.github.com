Install rvm. This is a very informative blog post about the steps to be made: http://www.unfoldingcode.com/2011/05/using-rvm-to-install-rails-31-on-ubuntu.html

When you try: gem install rails most certainly you will get "no such file to load -- zlib"
I wish you find a fast solution on this error. However what worked for me was to follow the steps from this response: http://stackoverflow.com/questions/1250329/has-anyone-tried-installing-ruby-rubygems-from-source-on-ubuntu-preferably-un/7126813#7126813

Why the blog post? Some weeks ago I bought myself a very good book: Rails 3 in Action. My learning progress can be checked here: https://github.com/valentinzberea/tracker. It's really a great experience going through this book. I like very much about it the way testing is treated. I've started the book project on Windows after rails was setup with http://railsinstaller.org/. Last night I've decided to continue work on a Ubuntu virtual machine.

More error messages:

"ERROR:  Error installing sqlite3-ruby" "sqlite3.h is missing."
I've found my answer here: http://stackoverflow.com/questions/3458602/sqlite3-ruby-install-error-on-ubuntu/3649005#3649005

"Could not find a JavaScript runtime." when you try "rails server"
sudo apt-get install nodejs

Somehow related: "Segmentation fault" when you try to install janus vim configuration
This worked for me:

rvm use 1.8.7 
cd ~/.vim/ruby/command-t
make clean
ruby extconf.rb
make

After you get the "segmentation fault" you will fix the command-t plugin then run gvim. That's it.
